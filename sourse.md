# 摘要
互联网行业近些年来的一种趋势是，重心逐渐由PC端移向移动端。各大互联网公司都在努力将自己PC时代的产品移植到移动端，最典型的一个例子就是王者荣耀团队放弃原来的PC游戏，投入全部力量研发王者荣耀这款现象级的移动手游；2017年淘宝双11成交额1682亿，90%贡献于移动端。移动端相较于PC端，一个显而易见的优点就是，移动端可以做到任何时刻都在线，不受时间和地点的限制，用户只需要带着手机，就可以应付生活中的绝大数场景。用户对移动端的依赖度越来越高，这也就是互联网公司持续在移动场景上不断发力的原因。互联网重心移向移动端所带来的最直接的好处就是用户只需要携带一部手机，就可以享受众多移动应用的便利。

移动互联网的发展给人们的生活带来巨大的便利，但令人遗憾的是，这些得到充分发展的移动场景都是市场规律使之。一些公益移动场景，比如本次毕业设计所关注的走失人口信息发布及寻找这一场景，因缺乏市场价值，所以发展缓慢。  虽然是市场规律，但还是十分遗憾。


现行的走失人口信息发布及寻找平台数量不多，服务质量总体偏差，且绝大多数仍停留在以网站为载体的场景下。首先这些网站的出发点是十分值得尊重的，但不可忽略的是，它们繁琐的流程、高学习成本、低效率的信息展示及检索，以及web应用固有的缺点，使得它们发挥的作用十分有限。另一方面，我国是发展中的人口众多、国情复杂的国家，人口走失以及拐卖现象时有发生，走失人群在社会上已经是一个不小的群体。结合上述移动端相较于PC端的优势，本毕业设计着力于研发出一个高效的走失人口信息发布寻找移动平台。走失人口这个弱势群体本应该享受到移动互联网高速发展带来的红利。

# 选题背景

## 庞大的失踪人口群体
2007年5月，凤凰卫视《社会能见度》栏目曾报道，“我国每年约有20多万儿童失踪，目前仍然没有放弃寻找的案件逾60万件”。这组数据来源于凤凰卫视。这仅仅是失踪儿童的数量，社会上还有大量的被拐卖的妇女和因智力残缺而导致的人口走失。此外我国还有众多的智力障碍患者和阿尔茨海默老人患者，他们和儿童一样，缺乏完全行为人能力，是走失的高位群体。这个群体加起来是一个非常庞大的群体

## 不完善的社会救助体系
近些年来我国不断在社会救助体系上投入人力物力，社会救助体系得以长足发展，但由于我国还处在发展中阶段，精力有限，难以在如此广阔的领土上构建一个完善的社会救助体系。所以救助站总体数量偏少，环境也不理想。

## 社会关注度低
人口走失现象多发于经济欠发达的地区，这些地区普遍存在信息流通缓慢现象，导致这一人群在社会上得到的关注较少。随着近年来，一些文艺作品，如《亲爱的》、《失孤》的出现，使得这一群体才逐渐走向公众视线，一些互联网巨头如百度阿里新浪纷纷在自己的核心产品上推送失踪人口信息，但这还不够，这个弱势群体还需要公民社会给予更多的关注。

# 国内外研究现状
国内现状
国内的走势人口信息发布寻找平台的发展可以归结为两个阶段和三个力量。第一个阶段是PC时代，平台都是以网站为载体，内容均以帖子的形式存在，为代表的是《宝贝回家寻子网》、《天涯寻亲网》。这些平台成立较早，平台的信息条目也比较多，但在信息展示以及信息检索方面存在严重短板，而且信息只能在平台内部流通，只有访问这个站点的人才可以看到信息，所以寻人效果并不高效。第二个阶段是移动互联网时代，这个阶段的载体是移动端应用，为代表的是《寻找你》、以及高德地图、支付宝、新浪微博推送的走失人口信息。移动端app有着良好的实时性。可以在第一时间通知附近或关注的用户。但目前移植到移动端的平台还非常少，且基本处在暂停服务的状态。高德地图、支付宝、新浪微博这些平台因其本身有自己的应用场景，所以在其平台上推送走势人口信息虽然高效，但是缺乏针对性。
三种力量分别是以政府为主导的公益性质运营的网站，为代表的是《中国走失人口档案库》、《全国救助寻亲网》。这些平台权威度较高，收集的信息条目也比较全。第二种力量是有实力比较强在某些方面有一定影响力的企业，为了承担社会责任而推出的平台。为代表的有中央电视台的《等着你》、高德的公安部走失人口信息发布平台。最后一种力量是民间的志愿者力量，部分采用公益化运营，部分采用商业化运营。为代表的是《宝贝回家寻子网》、《沈浩寻人网》等等。

国外现状
国外目前运行效果最好、最具有代表意义的的平台是《美国失踪儿童干预系统》，也就是我们常称的安珀系统（AMBER：America’s Missing: Broadcast Emergency Response）。安珀报警于1996年发展，最初使用“紧急报警系统”来发布少年儿童绑架案，紧急报警系统主要用于发布天气预报及重大紧急情况的警报。其后，安珀报警与许多组织、社区、政府机构合作，发展很快：2000年国会通过决议鼓励全美各地社区加入安珀报警、2001年全国儿童失踪中心加入推广安珀报警的努力、2002年联邦通信委员会为安珀报警背书、2003年国会通过法案，由司法部设立专门机构协调全国各地的安珀报警并提供联邦资金、2005年时，全美50个州都设立了安珀报警系统。，其影响之深远，从一组数字便可以看出：1990年时，全美只有62%的失踪儿童可以被找回，而如今，这一比例已经变为97.7%。

# 选题意义

关注弱势群体
交通的发展使得人们的出行越来越频繁，走失高危群体走失的现象越来越多，在社会救助体系未完全覆盖全社会时。能够早一点找回走失的人，就能减少一些损失。严重一点，在走失后的某段时间内未能找回，将会是不可逆转的严重后果。所以本次毕业设计选题的背景是目前社会上存在着大量由于走失或者被拐卖的人群，针对这些人群的信息发布和寻找平台的数量不多且都以公益性质的网站运营导致信息传递效果不佳。走失人口继续过着前途未卜的生活，寻找他们的亲人们继续受着身心的煎熬。本选题就是着力研究搭建一个高效的走失人口信息传递的平台。

检验大学学习成果
大学期间学校开设了《C语言程序设计》、《数据库原理》、《操作系统概论》、《数据结构》等计算机课程。课余时间本人还自学了IOS开发，所以本次毕业设计努力结合大学所学知识，搭建一个高效的走失人口信息发布、寻找平台，以检验本人的学习成果。

#系统分析
# 目前社会上运营的走失人口信息发布平台存在的问题大致分为以下几点：
1.1 发布走失人口信息流程复杂   
现有的为数不多的公益寻亲网站要么发布走失人口信息的流程十分繁琐，要么就干脆没有信息发布的入口，这些降低了寻人信息发布的成功率，也降低了其他公益志愿者参与的积极性。
1.2 平台没能较好的组织信息的展示  
部分平台在获得走失人口信息之后，并没有精准的将其展示在需要的人群面前，而是简单发布了之，也没有跟踪手段。使得信息发布之后便石沉大海。即简单采集了信息，却没有让这些信息流动起来。
1.3 平台缺乏智能的手段帮助降低寻人的难度  
部分平台在信息检索方式上十分粗旷，寻人者只能在众多的数据当中一条一条的检索。这种方式对寻找效率没有任何提高。直接限制了寻人的成功率。
1.4 缺乏移动端  
为数不多的公益寻人平台全都是采用网站的形式，网站的性质决定了寻人者无法时刻在线，另一方面，信息通知方面也是网站的薄弱点。没能充分利用移动智能设备的优点。
# 研究的基本内容与拟解决的主要问题
本平台研究的基本内容有三方面，其一是如何让走失人口信息的得到高效精准展示；其二是通过条件检索，人脸图片比对技术来降低寻人者获得期望的信息的难度。最后是对走失人口信息的动态跟踪，使得这些对寻人来说变得更加有价值。主要希望解决以下问题：  
2.1 实现一个步骤简洁的走失人口信息发布渠道
通过良好的产品及UI设计，实现一个流程流畅，操作简单但获取信息充分的一个走失人口信息发布的渠道，降低信息发布的学习成本，从而获得更多的走失人口信息，才能增加寻人成功的几率。
2.2 实现一个组织良好的走失人口信息展示板块  
走失人口信息的精准的展示给关注人是一个非常重要的部分，这也是目前社会上运营的公益寻人网站的薄弱环节。本平台将采用相关算法从寻人用户发布的寻人信息中提取相关信息，并着重向其展示平台计算出来的与寻人用户在寻找的走失人相似的信息。
2.3 提供基于条件选择以及人脸图片比对的检索功能  
在采集走失人口信息时做好信息分类的准备，在信息检索时可以根据条件分类来筛选信息，增加信息检索的有效性。同时还提供人脸图像比对的功能，帮助用户快速找到匹配度较高的走失用户的信息。
2.4 提供走失人口信息动态跟踪以及新动态推送关注人的功能
每一个发布在本平台的走失人口的信息并不将仅仅是一条消息本身，它会随着不同志愿者提供的有价值的信息的过程形成一个十分丰富动态跟踪，这些动态对走失人口的寻找十分有帮助。同时每一条动态更新都会推送到关注本条消息的用户，使得信息不停的在交换流通。
2.5 构建一个走失人口信息数据安全的平台
走失人口的信息发布会产生大量的用户隐私数据，这些隐私数据有助于走失人口的寻找，但同时也应该注意保护这些隐私数据，防止被不法之徒利用。同时要注意保护走失人口信息发布者的隐私，防止不法分子利用其心理特性来进行诈骗。

# 可行性分析
4.1 需求可行性分析
我们的社会上目前存在着大量的由于失智走失或者被拐卖的人群。对于由于智力障碍而走失的人群，社会救助体系尚未完成，且条件较差。这类走失人口如果信息发布得当，信息流通无障碍，家属能够在第一时间将其找回的几率还是很高的。对于被拐卖的妇女和儿童这类人群，如果有一个覆盖面很广的信息发布平台，对于他们的解救十分有帮助，而且，一个活跃的走失人口信息发布平台对拐卖儿童妇女的不法者本身就是一个极大的震慑，能够降低他们违法犯罪的几率。互联网高速发展的今天，走失人口这个弱势群体本应该享受到互联网技术带来的红利。
4.2 技术可行性分析  
移动应用从智能手机的流行开始就一直很热门，越来越多的应用从web端向移动端迁移。从2007年至今各个社区已经积累了大量的开发经验。编程语言、开发工具、各种第三方库、第三方服务、开发流程都有可参考的内容，且已经十分成熟。本平台所采用的基于C/S架构、基于Flask的web服务器框架、RESTful API 架构、JSON数据交互，IOS Development都有大量的开发人员使用，社区贡献了很多高价值的学习资料和开发经验。所以总体开发难度不大，难度主要集中在业务模型提取，并转化成具体的功能模块，还有如何高效精准的展示走失人口信息、动态跟踪、保护隐私数据安全的策略上。

# 系统总体设计
# 项目规划
根据上一章节的系统分析，本次毕业设计将本项目总体划分为五大功能模块，即用户账户管理模块、走失人口信息发布模块、走失人口信息展示模块、走失人口信息检索模块、走失人口信息交互模块。功能模块分类清晰、互相协同工作。确保系统高效运行。
# 用户账户模块。
用户账户管理模块是一个平台最基础的模块之一，是给用户提供服务的前提。由于本平台上发布的信息都是真人信息，容易被不法分子抓取利用，所以本平台除了提供常规的登录、注册、修改密码、修改个人信息等常规用户账户模块功能外，还提供了用户真人身份验证、以及用户信用积分模块。

首先没有进过身份认证的用户不能发布走失人口信息，这样可以保证平台上的信息的真实性；同时也不能查阅到一条走失人口的全部信息，这样可以保证发布在平台上的信息被乱用。

同时平台还引入信用积分制度，平台通过制定一系列策略并以积分来奖惩用户在平台上的行为。用户的积分对所有的用户公开，这样发布者在判断一个用户提供的线索是否有价值就有了基本的判断依据。同时如果用户的信用积分低于某个阈值时，就会被平台限制使用，而且平台还会保留相关证据，必要时可以提供给公安机关

# 走失人口信息发布模块
走失人口信息发布模块是本平台重要的信息来源（未来本平台还会从其他平台导入数据，同时本平台的数据也会输出给有公信力的平台）。这个模块十分重要，功能入口要放在最显眼的位置，而且采集信息的流程要尽可能的简单流畅，以降低用户的学习成本；同时在采集信息时要尽可能的全面，而且要设计好采集模型，以支持条件检索和机器自动匹配处理，大大提高用户检索信息的效率；在采集用户将要发布的信息完成之后，在真正发布之前，系统要将用户的信息模型化，去和数据库比对，然后反馈给用户最接近他将要发布的的若干个条目，让用户确认自己要找的人是否就在其中。避免信息被重复发布，以提高寻人的成功率。

# 走失人口信息展示模块
走失人口信息展示是平台发挥作用至关重要的一步。在采集了海量用户发布的信息之后，如何将其高效的展示出来是本次毕业设计的一个重点。反观现行运行的寻人平台，在获取到用户的信息之后，只是简单的在平台上发布了之，缺乏一系列的展示策略，使某个用户感兴趣的信息丢失在信息海洋之中。正是这些策略的缺失，才使得现行平台寻人成功率较低。

本平台的展示策略核心思想以某个用户为中心，通过采集用户在平台上的行为，综合分析出可能对该用户有价值的信息条目，并优先向用户展示这些信息，以提高信息的展示效率。
具体来说，对于一个发布了寻人信息的用户，平台会分析其发布的信息，然后在平台数据库去比对，优先向该用户展示与其发布的寻人信息吻合度较高的条目；对于一个没有发布寻人信息的用户，平台认为他可能是志愿者，那么平台将会根据其所在的地理位置优先展示其附近的走失信息，或者根据他的搜索纪录、浏览纪录来分析出对其有价值的走失信息，并优先展示。  


# 走失人口信息检索模块
除了高效的走失人口信息展示模块，平台还需要一个精准的走失人口信息检索模块，检索模块是用户获得对自己有价值信息的最快方式。因此检索模块的入口也应该放在最显眼的位置；同时检索的内容也要尽可能的多，以抓取尽可能多的符合内容。具体到本平台来说，检索的对象包括用户发布的寻人信息、用户提供的线索信息、用户的留言信息。  

在检索方式上要支持通用检索，即输入一个关键字来检索；分类检索，即平台提供不同角度，每个角度提供若干个选项值，用户根据自己的需要来组合筛选。

# 走失人口信息交互模块
走失人口信息交互模块是本次毕业设计最核心的设计。也是本平台与现行寻人平台的最大区别之处。
现行寻人平台在发布了信息之后，再有没有任何手段去跟踪信息的状态，信息没办法流动起来，堙灭在信息海洋中不能发挥任何价值。本平台在设计之处就认真了研究了同类平台的优缺点，所以在实现本平台时特别侧重走失人口信息交互模块的功能设计，努力使这些信息能够流动起来，发挥作用。
本平台在处理用户发布的寻人信息时，不是像传统的寻人平台一样把这些信息当作一个帖子，而是把它们当做一个数字虚拟实体，这样做的好处这些信息可以进行多种操作，而且随着不停的操作，信息本身可以被不断的被丰富。基于这一基础，本平台的信息交互式依靠下列6个子模块来实现的：
关注：
用户可以关注自己感兴趣的寻人案例，关注之后，案例就会放置在我的关注里面。这样做的好处是，自己关注的案例不会存在下次找不到情况，而这种情况经常出现在现行寻人平台里。
分享：
用户可以把平台上的寻人案例分享到各大社交平台，通过社交平台的影响力和传播力可以达到覆盖更多的人群的目的，信息得以有效的流通。
提供线索；
平台上的用户如果发现有价值的线索，还可以向寻人案例提供，平台会整合这些线索，并将其添加到寻人案例上，寻人案例的信息得以丰富，给案例发布者和志愿者的寻人提供有效的帮助。
推送：
推送分两类，第一类是用户在发布寻人案例之后的推送。对于刚刚走失的，平台会第一时间推送给以发布者为中心的一定地理范围内的用户，随着时间的推移，这个范围会逐渐扩大。对于有明确可能走失去向的，则重点推送到该地区的用户。
第二类是推送给某个案例的关注者，如果某个案例有了最新动态，比如某个用户提供了一条线索，或者人已经找到，都会推送给其关注者。
留言点赞
用户可以在寻人案例下面留言和点赞，这样做可以增加一些人文关怀，增加平台用户活跃度，从而吸引更多的志愿者进入平台，从而提升走失人口这个弱势群体的社会关注度。

# 基于协同过滤的智能匹配系统
协同过滤是一种基于一组属性相同的用户或项目进行的匹配，它根据类似用户(与目标用户兴趣一致的用户)的偏好信息产生对目标用户的推荐列表。协同过滤算法主要分为基于用户的协同过滤算法和基于项目的协同过滤算法。  
 基于用户的(User based)协同过滤算法是根据相邻用户的偏好信息生成对目标用户的推荐。它的前提这样一个假设:假如某些用户对某一类项目的打分比较接近，就认为他们对其它类项目的评分也比较接近。协同过滤匹配系统采用统计计算方式检索目标用户的类似用户，并根据类似用户对项目的评分来预测目标用户对指定项目的评分，最后筛选出相似度较高的前若干个相似用户的评分作为匹配结果，并反馈给用户。这种算法不仅计算简单且精确度较高，被现行的协同过滤匹配系统广泛采用。User-based协同过滤推荐算法的核心原理就是通过相似性度量方法计算出最接近目标的相邻集合，并将最相邻的评分结果作为匹配预测结果返回给用户

# 基于APNS的精准推送服务

苹果推送服务Apple Push Notification service (APNs) ，是苹果远程通知功能的核心，它为应用程序开发人员提供了一种强大，安全和高效的服务，可以将信息发送到iOS（以及间接的watchOS），tvOS和macOS设备。在用户设备上初次启动应用程序时，系统会自动在应用程序和APN之间建立经认可的加密且持久的IP连接。 通过此连接，应用可以执行设置以使其能够接收通知。
用于发送通知的另一半连接（即提供商服务器和APN之间的持久安全通道）需要开发者在在线开发人员帐户中进行配置并使用Apple提供的加密证书。 提供者是您部署和管理的服务器，配置为与APN一起工作。 图6-1显示了远程通知的传递路径。
通过在提供商和应用中完成推送通知设置，开发者提供商可以向APN发送通知请求。 APN向每个目标设备传送相应的通知有效载荷。 收到通知后，系统将有效负载传递到设备上的相应应用程序，并管理与用户的交互。
如果应用程序通知在设备开启但应用程序未运行的情况下到达，系统仍可显示通知。 如果设备在APN发送通知时关闭电源，则APN将保留通知并稍后再试。
APNS可以保证推送准确的推送到用户。所以开发者可以基于苹果提供的推送服务，制定自己的推送策略，可以保证信息传递的及时性，这也是移动app和web应用最明显的不同。
